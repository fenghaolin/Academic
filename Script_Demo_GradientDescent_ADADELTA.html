Demo of Gradient Descent vs. ADADELTA

Example 1: 1-Dimensional problem f(x)=x^2, with the known minimum at x=0.
ADADELTA does not need to specify learning rate alpha, since it is adaptive.
For Gradient Descent, if the alpha is a constant, it may never converge even for very simple convex function such as f(x)=x^2.
In the example, with alpha=1 and x0=1, the Gradient descent keeps bouncings between 1 and -1 since its gradient=2x.
x0=1;
f=@(x) x^2;
grad_f=@(x, fx) 2*x;
tic;
[obj_A,x_A,grad_A,flag_A,iter_A]=adadelta(f,grad_f,[],x0);
t_A=toc;
tic;
[obj_G,x_G,grad_G,flag_G,iter_G]=ML_GradientDescent(f,grad_f,[],x0);
t_G=toc;
fprintf('ADADELTA used %.4f sec, %d iterations, (obj,x)=(%.4e,%.4e)\n',t_A,iter_A,obj_A(iter_A+1),x_A(iter_A+1));
ADADELTA used 0.0055 sec, 12 iterations, (obj,x)=(7.3817e-11,8.5917e-06)
fprintf('Gradient Descent used %.4f sec, %d iterations, (obj,x)=(%.4e,%.4e)\n',t_G,iter_G,obj_G(iter_G+1),x_G(iter_G+1));
Gradient Descent used 0.0031 sec, 100 iterations, (obj,x)=(1.0000e+00,1.0000e+00)
figure(1)
subplot(2,2,1);plot(x_A(1:iter_A+1),'*-'); title('x evolution of ADADELTA')
subplot(2,2,2);plot(obj_A(1:iter_A+1),'.-'); title('obj evolution of ADADELTA')
subplot(2,2,3); plot(x_G(1:iter_G+1),'*-');title('x evolution of GS, alpha=1')
subplot(2,2,4); plot(obj_G(1:iter_G+1),'.-'); title('obj evolution of GS,alpha=1')
suptitle('f(x)=x^2, x0=1.')
Example 2: 2-Dimensional problem f(x)=x^T A x+ B X, A=[2,1;1,1]; B=[1,-1];
A=[2,1;1,1]; B=[1,-1];
f=@(x) x*A*x'+B*x';
grad_f=@(x,fx) 2*x*A'+B;
x0=[1,1];
tic;
[obj_A,x_A,grad_A,flag_A,iter_A]=adadelta(f,grad_f,[],x0);
t_A=toc;
tic;
[obj_G,x_G,grad_G,flag_G,iter_G]=ML_GradientDescent(f,grad_f,[],x0);
t_G=toc;
fprintf('ADADELTA used %.4f sec, %d iterations, obj=%.4e,x=(%.4e,%.4e)\n',t_A,iter_A,obj_A(iter_A+1),x_A(iter_A+1,1),x_A(iter_A+1,2));
ADADELTA used 0.0078 sec, 79 iterations, obj=-1.2500e+00,x=(-9.9995e-01,1.4999e+00)
fprintf('Gradient Descent used %.4f sec, %d iterations,obj=%.4e,x=(%.4e,%.4e)\n',t_G,iter_G,obj_G(iter_G+1),x_G(iter_G+1,1),x_G(iter_G+1,2));
Gradient Descent used 0.0028 sec, 100 iterations,obj=1.3376e+126,x=(6.0804e+62,3.7579e+62)

If we change alpha for Gradient Descent
for alpha=0.1:0.1:1
tic;
[obj_G,x_G,grad_G,flag_G,iter_G]=ML_GradientDescent(f,grad_f,[],x0,alpha);
t_G=toc;
fprintf('Gradient Descent, alpha=%.2f, used %.4f sec, %d iterations,obj=%.4e,x=(%.4e,%.4e)\n',alpha,t_G,iter_G,obj_G(iter_G+1),x_G(iter_G+1,1),x_G(iter_G+1,2));
end

We can see that when alpha is small, it takes many iterations to converge. When alpha is too large, it does not converge at all.
