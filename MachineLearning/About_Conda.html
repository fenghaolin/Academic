Why Anaconda

  • 部分内容参考了这里 和Python 数据科学手册
  1. Anaconda集成了大部分科学计算和数据科学中需要用到的Python包，因此你几乎可以在安装后直接用于数据科学、机器学习、科学计算等领域。
  2. 同时，利用conda，Anaconda提供了容易操作的环境管理方式。这里说的环境是独立的、互不干扰的开发环境。

How to work with conda

  1. 下载并安装（下载完应该从官网 查看一下散列值是否一致，以确保完整性）, windows下可以只选为当前用户安装（或为所有用户安装，但需要管理员权限，没有必要）
  2. 安装好之后， 更新

conda --version
conda update conda

对于包的下载，我们可以先设置离我们较近的镜像地址，否则conda在下载包的时候速度可能不快。清华TUNA镜像 有列出Anaconda仓库的镜像，将当中的加入conda的配置即可解决这个问题。

添加Anaconda的清华镜像(TUNA)

conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
conda config --set show_channel_urls yes

如要看现在的channels，可以这样： conda config --show channels

  1. 环境管理
  • 安装完成之后，默认的环境叫base，可以在上面安装额外的包，并指定从哪里安装，例如安装第三方分词库jieba

    • conda install -c conda-forge jieba

  • 一般，会为项目创立自己的环境，以免各个项目依赖的包发生版本冲突

2.1 创建新的环境（以安装一个data-science的环境为例，指定python版本为3.9

conda create --name data-science python=3.9

  • 如果不指定python版本，就是以默认系统的python版本创建环境
  • 如果对各种要用的package有要求，就加上对于的package的requirements.txt:

conda crate --name data-science --file requirements.txt

2.2 激活环境

conda activate data-science

2.3 退出环境 (windows环境下使用deactivate)

conda deactivate

2.4 删除环境
如果不想要这个名为data-science的环境，可以通过以下命令删除这个环境。

conda remove --name data-science --all

之后可以查看剩下的环境

conda info -e

  1. 关于包管理
    我们使用conda进行第三方包的安装、卸载和更新。
  • 查看已安装的包 (使用这条命令来查看在当前环境中，已安装的包和对应版本)

conda list

下面我们以pandas包为例

  • 查找可安装的包
    • 我们可以通过search命令检查pandas这个包是否可以通过conda来安装

    • conda search pandas

    • 命令返回了这个包的信息，所以是可以通过conda安装的
    • 安装pandas conda install pandas
  • 更新包: 以 pandas 为例

conda update pandas

  • 卸载pandas包

conda remove pandas

对其他包也是这样管理
